자바 기반의 빌드 툴

자카르타 프로젝트 
자바기반의 오픈 소스 소프트웨어를 개발하여 무료 배포하기 위한
목적의 프로젝트이다. 아파치 재단이 지원하며 톰캣, 앤트도 자카르타의
산출물이다. 
 
os와 상관없이 사용가능. ant configuration파일이 xml로 되어있어
어플리케이션 구조에 맞게 적용하기에 편리하다 

빌드 << 사용자의 컴퓨터에 설치되어 실행되는 단계. 

ANT 기능
자바소스코드 컴파일 
jar war ear zip 파일 생성

자바의 jar tool을 이용된 압축(아카이브)파일. 
어플을 쉽게 배포하고 동작시킬 수 있도록 관련된 리소스,속성파일들을
패키징해주는 것이 주 역할이다. 


.jar - java archive (아카이브 : 저장, 압축의 의미)
>>class와 같은 java 리소스, 속성파일, 라이브러리 및 액세서리 파일 포함
자바 프로젝트를 압축한 파일 같은 개념, 플랫폼에 귀속되는점 빼면 zip과 비슷하다
jdk에 포함하고있는 jre만 가지고도 실행 가능


.war - web application archive
>>servlet /jsp 컨테이너에 배치 할 수 있는 웹 어플리케이션 압출 파일 포맷이다. 
jsp, servlet, jar, class, xml, html, javaScript 등 servletContext관련 파일들로 패키징
되어 있다. 웹 응용 프로그랩을 위한 포맷이으로 웹 관련 자료들만 포함하고 있으며
웹 어플리케이션을 쉽게 배포하고 테스트 할 수 있다. 
jar와 달리 WEB-INF, META-INF 디렉토리 등 사전에 정의된 구조를 사용하며
war파일이 실행되려면 Tomcat, Weblogic, Websphere등의 웹서버나 
웹컨테이너(WAS)가필요하다 
war파일도 java의 java -jar를 이용해 생성하는 jar파일의 일종으로 
웹 어플리케이션 전체를 패키징하기 위한 jar파일의 개념이다. 

.ear - enterprise archive
java ee(enterprise edition)에 쓰이는 파일 형식으로 한 개 이상의 모듈을 
단일 아카이브로 패키징하여 어플리케이션 서버에 동시에 일관적으로 
올리기 위해 사용하는 포맷이다. 

.xml -eXtensible Markup Language
데이터들을 쉽게 구조화하여 전달하기 위한 문서 형식이다. 
구조화된 문서.
메모장으로 열리고 쉽게 수정, 열람이 가능하다. 
비슷한 기능으로 json 파일 형식이 있다. 
xml 선언 <?xml version="1.0" encoding="UTF-8" ?>
사람과 기계가 동시에 읽기 편한 구조이다. 
html처럼 데이터를 보여주기 위한 것이 아닌 저장과 전달이 목적이다. 
html처럼 태그가 미리 정의 되어 있지 않고 xml은 사용자가
직접 태그를 정의한다. 

다른 시스템끼리 다양한 종류의 데이터를 교환하게 해준다. 
새로운 태그를 만들어 추가해도 계속 동작하므로 확장성이 좋다
텍스트 데이터 기반의 언어이며 유니코드 문자로만 이루어져 있다.
